OSPFv3 Authentication / Encryption

OSPFv3 uses IPsec for Authentication and Encryption

IPsec Encapsulation Types

  • AH - Authentication Header
  • ESP - Encapsulating Security Payload

The AH protocol provides a mechanism for Authentication only

The AH algorithms are HMAC-MD5 or HMAC-SHA

The ESP protocol provides data Confidentiality and/or Authentication

The ESP algorithms are DES or 3DES or AES-CBC

Authentication Header

Authentication Header

AH requisits

  • SPI
  • Algorithm
  • KEY

SPI should be unique per interface

SPI range is 256~4294967295

SPI in the two router should be the same

!

interface Ethernet0/0

ipv6 enable

ipv6 ospf authentication ipsec spi 256 md5 202CB962AC59075B964B07152D234B70

ipv6 ospf 1 area 1

!

Symmetric Encryption Algorithms

Algorithms:

  • DES - Data Encryption Standard
  • 3DES - Triple Data Encryption Standard
  • AES - Advanced Encryption Standard (6 times faster than the 3DES)

AES

  • AES is a block cipher
  • The key size can be 128/192/256 bits
  • The industry standard

AES Modes:

  • ECB - Electronic Code Book (Suitable for encrypting small messages)
  • CBC- Cipher Block Chaining (Better suited for encrypting large messages)
  • OFB - Output FeedBack
  • CTR -Counter
