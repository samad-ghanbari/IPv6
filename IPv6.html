OSPFv3 Authentication / Encryption

OSPFv3 uses IPsec for Authentication and Encryption

IPsec Encapsulation Types

  • AH - Authentication Header
  • ESP - Encapsulating Security Payload

The AH protocol provides a mechanism for Authentication only

The AH algorithms are HMAC-MD5 or HMAC-SHA

The ESP protocol provides data Confidentiality and/or Authentication

The ESP algorithms are DES or 3DES or AES-CBC

Authentication Header

Authentication Header

AH requisits

  • SPI
  • Algorithm
  • KEY

SPI should be unique per interface

SPI range is 256~4294967295

SPI in the two router should be the same

!

interface Ethernet0/0

ipv6 enable

ipv6 ospf authentication ipsec spi 256 md5 202CB962AC59075B964B07152D234B70

ipv6 ospf 1 area 1

!

IPv6 Automatic 6to4 Tunneling Drawbacks

IPv6 6RD Tunneling - Rapid Deployment

  • ISPs Uses Its Own Unique IPv6 Prefix For 6RD
  • Relay Router Belongs To The ISP
  • Communication With Native IPv6 Network
  • ISPs Can Have Multiple Border-Relay-Router

6RD Steps

  • Specify The IPv6 Prefix
  • Specify Virtaul IPv4 Subnet
  • Specify The BR Router
  • Route The Prefix To The Tunnel Interface
  • Route Default To The BR Router

MPLS Network Tunnel

  • 6PE
    • IPv4 MPLS core
    • PE Routers Are Dual-Stack
    • Protocol Is MP-BGP
    • IPv6 Traffic On Global
  • 6VPE
    • IPv4 MPLS core
    • PE Routers Are Dual-Stack
    • Protocol Is MP-BGP
    • IPv6 Traffic On VRF (VPN-Instance)

6PE

  • PE routers By Using MP-BGP and IPv4-LSP Will Exchange The IPv6-Prefixes
  • The Next-Hop Address For IPv6-Prefixes Will Be IPv4
  • Next-Hop For IPv6-Prefixes Will Embed The IPv4 Address Of The Peer-Router
  • Next-Hop Will Be Like ::FFFF:IPv4
  • Next-Hop Address Will Be Invalid Unless MP-BGP Sends Peer The Label
  • IPv6 Traffic Sends By Double-MPLS-Tag
